# Code of Conduct

As contributors and maintainers of **JoyCode Documentation**, we are committed to fostering a welcoming, respectful, and collaborative environment. This project is part of the wider **JoyCode Studios** ecosystem, and all participants are expected to uphold the standards outlined below.

This Code of Conduct applies to all project spaces, including issues, pull requests, discussions, documentation, and any other interactions related to the project.

---

## 1. Our Standards

We aim to create an environment where everyone feels respected and able to contribute. Examples of behaviour that supports this goal include:

- Being considerate and constructive in discussions  
- Offering helpful feedback without hostility  
- Respecting differing viewpoints and experiences  
- Accepting responsibility for mistakes and learning from them  
- Focusing on what is best for the project and its users  
- Using inclusive and welcoming language  

Examples of unacceptable behaviour include:

- Personal attacks, insults, or disrespectful comments  
- Harassment, intimidation, or discrimination of any kind  
- Dismissing or belittling others’ contributions  
- Sharing private information without permission  
- Disruptive or intentionally unproductive behaviour  

---

## 2. Responsibilities of Maintainers

Project maintainers are responsible for:

- Clarifying and enforcing this Code of Conduct  
- Reviewing contributions fairly and respectfully  
- Taking appropriate action when unacceptable behaviour occurs  
- Ensuring the project remains a safe and collaborative space  

Maintainers have the right to remove comments, close issues, reject contributions, or ban individuals who violate this Code of Conduct.

---

## 3. Scope

This Code of Conduct applies:

- Within all project repositories  
- In issue trackers, pull requests, and discussions  
- In any public or private communication related to the project  
- In any space where individuals represent the project or JoyCode Studios  

Representation includes using an official project email, posting as a project maintainer, or acting as a community representative.

---

## 4. Reporting Issues

If you experience or witness behaviour that violates this Code of Conduct, please report it by opening a private issue or contacting a project maintainer directly.

Reports will be handled discreetly and respectfully.  
Information will only be shared with those who need to know in order to address the situation.

---

## 5. Enforcement

Maintainers will investigate all reports and determine appropriate action, which may include:

- A warning  
- Temporary or permanent removal from the project  
- Reversal of contributions if necessary  
- Restriction from future participation  

All decisions will be made with fairness, consistency, and the best interests of the project in mind.

---

## 6. Attribution

This Code of Conduct is inspired by common open‑source community guidelines and adapted specifically for the JoyCode Studios ecosystem.

---

## 7. Commitment to Openness

JoyCode Documentation is MIT‑licensed and open to everyone.  
By participating, you help strengthen a shared ecosystem built on clarity, respect, and collaboration.

Thank you for helping make this project a welcoming place for all contributors.
