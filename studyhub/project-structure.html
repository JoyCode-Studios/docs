Folder Breakdown

public/

Contains static assets served directly without processing. Examples:

  • icons
  • manifest files
  • static images

src/

The main source folder containing all React code.

src/components/

Reusable UI components shared across pages. Examples:

  • Navbar
  • Sidebar
  • NoteCard
  • TaskItem
  • SubjectTag

src/pages/

Each StudyHub feature has its own page component. Examples:

  • Dashboard.jsx
  • Notes.jsx
  • Tasks.jsx
  • Subjects.jsx
  • Timetable.jsx

src/hooks/

Custom React hooks for shared logic. Examples:

  • useLocalStorage.js â€” wrapper for reading/writing localStorage
  • useTheme.js â€” theme toggle logic

src/storage/

Contains helper modules for managing StudyHubâ€™s localStorage keys. Examples:

  • notes.js
  • tasks.js
  • subjects.js
  • timetable.js

src/styles/

Global and componentâ€‘specific CSS files. Examples:

  • global.css
  • dashboard.css
  • notes.css

App.jsx

The root component that defines routing and layout.

main.jsx

Entry point that mounts the React app and initialises global providers.

Build & Config Files

index.html

The base HTML file used by Vite to inject the React app.

vite.config.js

Vite configuration file for plugins, aliases, and build settings.

package.json

Contains dependencies, scripts, and project metadata.

Why This Structure?

This layout keeps StudyHub:

  • modular
  • easy to maintain
  • scalable for future features
  • consistent with modern React best practices
