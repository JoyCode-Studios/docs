Timetable (Monday‚ÄìFriday)

The StudyHub Timetable is a simple weekly planner that lets you assign subjects to each weekday. It is designed to help you visualise your study schedule and connect subjects to your dashboard overview. All timetable data is stored locally using localStorage.

How the Timetable Works

The timetable consists of five columns ‚Äî one for each weekday:

  • Monday
  • Tuesday
  • Wednesday
  • Thursday
  • Friday

Each day contains a list of classes. You can add as many classes as you need, and each class is linked to a subject from your Subjects Manager.

Adding a Class

To add a class to a day:

  • Select the day (e.g., Monday)
  • Click Add Class
  • Choose a subject from your subject list
  • Optionally add a time or note

The class will appear immediately in the timetable grid.

Editing or Removing a Class

Each class entry includes options to:

  • Edit ‚Äî change the subject or details
  • Delete ‚Äî remove the class from that day

Changes are saved instantly.

LocalStorage Key

The timetable is stored under: 

studyhub_timetable

The value is an object containing arrays for each weekday: 

{
  monday: [...],
  tuesday: [...],
  wednesday: [...],
  thursday: [...],
  friday: [...]
}

Each class entry contains:

  • id ‚Äî unique identifier
  • subjectId ‚Äî links to a subject
  • time ‚Äî optional time label
  • note ‚Äî optional description

Dashboard Integration

The dashboard uses your timetable to show:

  • today‚Äôs classes
  • upcoming subjects
  • the days of the week from Monday to Friday

If you delete a subject, timetable entries referencing it will show as ‚ÄúUnassigned.‚Äù

Limitations

Because the timetable uses localStorage:

  • it does not sync across devices
  • clearing browser data removes all classes
  • timetable entries are tied to the specific browser and device
